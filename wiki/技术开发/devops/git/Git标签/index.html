Git标签

Git 可以对某一时间点上的版本打上标签，经常用于某个软件版本（比如v1.0等）。

一、列出标签

二、新建标签

Git 的标签分为两类：

  • 轻量级(lightweight)的标签，指向特定提交的引用，就像个不会变化的==分支==。
  • 含附注(annotated)的标签，实际上是存储在仓库中的一个==独立对象==。

四、签署和验证标签

使用私钥签署(GPG)标签，需要使用对应公钥进行验证。

五、推送标签到远端仓库

添加完标签后，需要推送到远程仓库，其他人才能获取到标签信息。
